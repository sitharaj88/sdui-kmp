# =============================================================================
# sdui-kmp sample-android baseline profile (hand-curated, stub)
# =============================================================================
#
# Format: ART human-readable baseline profile. One class / method per line.
# `H` = method should be precompiled to native (hot).
# `S` = startup method (referenced during the first frame after process start).
# `P` = post-startup method (referenced during the first 5 seconds after launch).
#
# Why hand-curated, not generated via macrobenchmark:
#
# `androidx.benchmark:benchmark-macro-junit4` and the `androidx.baselineprofile`
# Gradle plugin both require AGP >= 8.7 *and* a Kotlin / Compose Multiplatform
# combination that has not yet stabilised on Kotlin 2.1.0 + Compose MP 1.7.3
# in this repo. Wiring the macrobenchmark module today produces a tangle of
# ksp / hilt / hilt-android transitive resolution issues that are not worth
# fighting until the next Kotlin bump.
#
# Until then, this file is a manual approximation: the protocol decoder hot
# path, the SDUI runtime renderer entry points, the http transport, and the
# core widget composables that fire on first screen render.
#
# Regenerate via the macrobenchmark module once the build catches up — see
# docs/ops/baseline-profiles.md for the full procedure. The file is bundled
# into release APKs by AGP automatically when located at
# `src/main/baseline-prof.txt`; no Gradle wiring is required.
